## Objective 1
### "크리에이터가 여기서만 받을 수 있는 피드백을 위해 매일 돌아오는 플랫폼을 만든다"

바이브폴리오의 핵심 차별점은 미슐랭 평점, 성장센터, 시크릿 코멘트 등 **깊이 있는 피드백 시스템**이다.
단순 '좋아요' 플랫폼이 아니라, 크리에이터가 실질적으로 성장하는 공간이 되어야 한다.

### Key Results

**KR 1.1** — 성장센터(`/growth`)에 등록되는 프로젝트 수를 **월 50건 이상**으로 끌어올린다
- 현재 추정: 월 10건 미만
- 측정: `SELECT COUNT(*) FROM Project WHERE is_growth_requested = true AND created_at >= 이번달_시작`
- 70% 달성(35건)이면 성장센터가 활성 커뮤니티로 자리잡은 것

**KR 1.2** — 미슐랭 평점이 달린 프로젝트 비율을 전체의 **30% 이상**으로 높인다
- 현재 추정: 10% 미만
- 측정: `ProjectRating` 존재 프로젝트 수 / 전체 `Project` 수
- 미슐랭 평점은 4개 축(기획력, 완성도, 독창성, 상업성) 평가 — 플랫폼 핵심 차별점

**KR 1.3** — 프로젝트당 평균 코멘트 수를 **3개 이상**으로 늘린다
- 현재 추정: 1개 미만
- 측정: 전체 코멘트 수 / 전체 프로젝트 수
- 코멘트 = 커뮤니티 활성도의 가장 직접적인 지표

### Initiatives
- [ ] 피드백 보상 차등화: 미슐랭 평점 작성 시 +15P (현재 코멘트 +5P만 존재)
- [ ] 성장센터 온보딩 UX 개선 (피드백의 가치를 감성적으로 전달)
- [ ] 미슐랭 평점 작성 UI 간소화 (빠른 평가 모드 추가)
- [x] ~~AI 피드백 요약~~ (Gemini 1.5 Flash로 코멘트+평점 3줄 요약 생성, 프로젝트 모달에 표시)
- [ ] 주간 피드백 리포트 자동 알림 (프로젝트 등록자에게 요약 발송)
- [ ] 우수 피드백 작성자 하이라이트 (커뮤니티 인정 시스템)

## Objective 2
### "AI가 크리에이터에게 딱 맞는 기회를 알아서 찾아주는 지능형 플랫폼으로 진화한다"

채용/공모전 정보는 크리에이터가 바이브폴리오를 방문하는 강력한 이유 중 하나다.
HTML 파싱에 의존하던 불안정한 크롤링을 AI 기반으로 전환하여, 양질의 기회 정보를 안정적으로 제공한다.

### Key Results

**KR 2.1** — 공모전 크롤링 성공률을 **90% 이상**으로 끌어올린다
- 현재 추정: 30-50% (Wevity HTML 파싱 불안정)
- 측정: `crawl_logs`에서 `status='success' AND items_added > 0` 비율
- Tavily AI 크롤링이 PRIMARY이므로 HTML 깨짐에 영향받지 않아야 함

**KR 2.2** — 채용/공모전 페이지의 활성 항목을 **주간 50건 이상** 유지한다
- 측정: `SELECT COUNT(*) FROM recruit_items WHERE is_active = true AND date >= now()`
- 사용자가 방문할 때마다 신선한 기회 정보가 있어야 재방문 동기 부여

**KR 2.3** — 크롤링 공모전 정보의 **마감일 정확도 70% 이상** 달성
- 현재: '확인 필요', '검색 결과' 등 부정확한 날짜 다수
- 측정: `date` 필드가 유효한 `YYYY-MM-DD`이고 미래 날짜인 항목 비율

### Initiatives
- [x] ~~Tavily AI 기반 공모전 크롤러 도입~~ (`crawlContestAI` — 5개 쿼리 병렬)
- [x] ~~관리자 일괄 승인 기능~~ (크롤링 → 승인 병목 해소)
- [x] ~~LLM 후처리로 마감일 정확도 향상~~ (Gemini 1.5 Flash, 크롤링 시 '확인 필요' 날짜 자동 추출)
- [x] ~~사용자 관심 분야 기반 맞춤 공모전 추천 알림~~ (`interests` ↔ `category_tags` 매칭, 승인 시 자동 알림)
- [ ] 크롤링 소스 확대: 링커리어, 캠퍼스픽 API 연동 조사
- [ ] 크롤링 헬스 모니터링 (items_added=0이면 관리자 알림)

## Objective 3
### "어떤 순간에도 빠르고 안정적인 경험을 제공하여 크리에이터의 신뢰를 쌓는다"

느린 로딩, 접속 불가 같은 경험은 플랫폼 신뢰를 무너뜨린다.
관리자 페이지 1주일 접속 불가 사건(2026-02-13)을 교훈 삼아, 안정성과 성능을 체계적으로 관리한다.

### Key Results

**KR 3.1** — 홈 페이지 LCP(Largest Contentful Paint)를 **2.5초 이내** 달성
- 측정: Vercel Analytics / Chrome Lighthouse
- 모바일 기준, 3G 네트워크 시뮬레이션에서도 체감 가능

**KR 3.2** — 100개 이상 프로젝트 스크롤 시 **프레임 드롭 없이 60fps** 유지
- 측정: Chrome DevTools Performance 탭
- LazyImageCard 도입으로 기반 마련 완료

**KR 3.3** — 프로덕션 장애(접속 불가, 핵심 기능 오류)를 분기 내 **0건** 유지
- 측정: Vercel 배포 로그 + 사용자 보고
- middleware 인증 사고 재발 방지 체계 포함
