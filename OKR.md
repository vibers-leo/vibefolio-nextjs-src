# 바이브폴리오 (Vibefolio) OKR

> **작성일**: 2026년 2월 12일
> **프로젝트**: AI 창작자 포트폴리오 플랫폼
> **기간**: 2026 Q1 (1분기)

---

## 🎯 Objective (도전적 목표)

**"AI 창작자들이 자신의 작품을 자랑스럽게 전시하고, 더 많은 사람들에게 발견되며, 다시 찾고 싶은 플랫폼을 만든다"**

단순히 "기능 구현"이 아닌, AI 창작자들이 자신의 창작물을 세상에 보여주고 인정받을 수 있는 무대를 제공하는 것이 목표입니다. 비핸스처럼 전문적이면서도, AI 창작물에 특화된 커뮤니티를 구축하여 창작자들이 가슴 뛰는 경험을 하도록 만듭니다.

---

## 📊 Key Results (핵심 결과)

### KR 1: 사용자 참여도 - 월간 활성 사용자(MAU) 1,000명 달성
- **현재 상태**: 프로토타입 단계, 실제 사용자 데이터 없음
- **목표**: 1,000명의 월간 활성 사용자
- **측정 방법**: Supabase Analytics를 통한 고유 사용자 추적
- **도전 포인트**: 70% 달성(700명)만 해도 성공적인 시작

### KR 2: 콘텐츠 품질 - 업로드된 프로젝트 평균 조회수 100회 이상 달성
- **현재 상태**: 조회수 카운트 기능 미구현
- **목표**: 프로젝트당 평균 조회수 100회 이상
- **측정 방법**: 전체 조회수 / 전체 프로젝트 수
- **의미**: 단순히 많이 올리는 게 아니라, 실제로 사람들이 보고 관심 갖는 콘텐츠

### KR 3: 사용자 충성도 - 주간 재방문율 40% 달성
- **현재 상태**: 재방문 추적 기능 없음
- **목표**: 한 번 방문한 사용자 중 40%가 7일 이내 재방문
- **측정 방법**: (주간 재방문 사용자 수 / 전주 신규 사용자 수) × 100
- **의미**: 단순 구경이 아닌, 다시 찾고 싶은 플랫폼인지 측정

### KR 4: 사용자 경험 - 플랫폼 로딩 속도 2초 이내 & 사용자 만족도 4.5/5.0 달성
- **현재 상태**: 성능 최적화 진행 중 (이미지 로딩, Supabase 쿼리 최적화)
- **목표**:
  - 메인 페이지 초기 로딩 시간 2초 이내
  - 사용자 만족도 설문 평균 4.5/5.0
- **측정 방법**:
  - Lighthouse Performance Score
  - 사용자 피드백 설문조사
- **의미**: 빠르고 쾌적한 UX 제공

## 🚀 Initiatives (실행 방안)

> **KR 달성을 위한 구체적인 작업 목록 (코드 레벨)**

### Phase 1: 핵심 기반 구축 (1-2주)
- [ ] **Supabase 데이터베이스 완전 연동**
  - users, projects, likes, views 테이블 설계 및 구축
  - RLS(Row Level Security) 정책 설정

- [ ] **사용자 인증 시스템 구현**
  - NextAuth.js 또는 Supabase Auth 통합
  - 소셜 로그인 (Google, GitHub) 구현
  - 프로필 관리 페이지 구축

- [ ] **조회수 카운트 시스템 구현**
  - 프로젝트 클릭 시 views 테이블에 기록
  - 중복 조회 방지 로직 (IP 기반 또는 세션 기반)
  - 실시간 조회수 표시

### Phase 2: 사용자 인터랙션 강화 (2-3주)
- [ ] **좋아요 & 북마크 기능 구현**
  - 좋아요 버튼 UI/UX 개선
  - 북마크 컬렉션 기능 추가
  - 좋아요한 프로젝트 목록 페이지

- [ ] **댓글 시스템 구현**
  - 프로젝트 상세 페이지에 댓글 기능
  - 실시간 댓글 업데이트 (Supabase Realtime)
  - 댓글 알림 기능

- [ ] **프로필 & 포트폴리오 페이지**
  - 사용자별 프로필 페이지 구축
  - 업로드한 프로젝트 갤러리
  - 팔로우/팔로워 시스템 기초 구현

### Phase 3: 발견성 향상 (3-4주)
- [ ] **검색 기능 구현**
  - 프로젝트 제목, 설명, 태그 기반 검색
  - 사용자 이름 검색
  - 자동완성 기능

- [ ] **무한 스크롤 구현**
  - Intersection Observer를 활용한 lazy loading
  - 페이지네이션 대신 무한 스크롤 적용

- [ ] **정렬 & 필터링 강화**
  - 인기순 (좋아요 많은 순)
  - 최신순
  - 조회수 순
  - 카테고리별 필터링 개선

### Phase 4: 성능 최적화 & 분석 (지속적)
- [ ] **성능 최적화**
  - Next.js Image 컴포넌트 활용한 이미지 최적화
  - Lazy loading 적용 (이미지, 컴포넌트)
  - Supabase 쿼리 최적화 (인덱싱, 필요한 컬럼만 select)
  - Code splitting & Dynamic import

- [ ] **사용자 분석 시스템 구축**
  - Google Analytics 또는 Vercel Analytics 연동
  - MAU, 재방문율, 조회수 등 지표 추적
  - 사용자 행동 분석 (어떤 카테고리를 많이 보는지 등)

- [ ] **관리자 대시보드 개선**
  - 전체 통계 시각화 (사용자 수, 프로젝트 수, 인기 프로젝트)
  - 프로젝트 승인/거부 기능
  - 사용자 관리 기능

### Phase 5: 커뮤니티 기능 (장기)
- [ ] **알림 시스템**
  - 좋아요, 댓글, 팔로우 알림
  - 실시간 알림 (Supabase Realtime)

- [ ] **소셜 기능**
  - 팔로우/팔로워 시스템 고도화
  - 활동 피드
  - 추천 창작자

## 🔍 Self-Correction (자체 검토)

### ❌ 단순 '할 일 목록'인가?
**NO.** 이 OKR은 단순히 "기능 구현"에 그치지 않고, **사용자에게 주는 가치**에 집중하고 있습니다.

- **Bad 예시**: "좋아요 기능 만들기" (단순 Output)
- **Good 예시**: "좋아요를 통해 프로젝트 평균 조회수 100회 달성" (Outcome)

### ✅ 비즈니스 가치 창출인가?
**YES.** 각 KR은 비즈니스 성과와 직접 연결됩니다:

1. **MAU 1,000명**: 플랫폼의 생존 가능성 증명
2. **평균 조회수 100회**: 콘텐츠 품질과 사용자 참여도 증명
3. **재방문율 40%**: 지속 가능한 성장 가능성 증명
4. **만족도 4.5/5.0**: 제품-시장 적합성(Product-Market Fit) 검증

### 📈 70% 달성 시나리오
- MAU 700명: 초기 사용자 기반 확보 성공
- 평균 조회수 70회: 콘텐츠 소비가 활발히 일어남
- 재방문율 28%: 약 1/3 사용자가 다시 방문 = 긍정적
- 만족도 4.2/5.0: 사용자들이 대체로 만족

이 정도만 달성해도 "성공적인 1분기"라고 할 수 있습니다!

## 📌 핵심 원칙 (팀 공유)

1. **Output이 아닌 Outcome에 집중**
   - "댓글 시스템 만들었다" (X)
   - "댓글을 통해 프로젝트당 평균 인터랙션 5회 증가" (O)

2. **측정 가능한 지표 우선**
   - 모든 기능에는 측정 가능한 성과 지표가 있어야 함
   - 매주 진행 상황 체크

3. **사용자 가치 중심**
   - "우리가 만들고 싶은 것" (X)
   - "사용자가 필요로 하는 것" (O)

4. **70% 달성해도 성공**
   - 도전적인 목표이기 때문에 100% 달성하지 못해도 괜찮음
   - 중요한 건 방향성과 실행력
